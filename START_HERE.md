## 🧠 The Core Principle

All these documents express one idea:

```
Knowledge → Decisions → Execution → Validation
```

That's it. Everything else is just **naming and organizing** those layers.

- **Knowledge:** Research findings (Reddit, competitors, tech insights)
- **Decisions:** Architecture decisions (why RLS? why Playwright?)
- **Execution:** Code + documentation (what you're building)
- **Validation:** Tests + QA (proof you did what you planned)

The 9-layer structure (00_SYSTEM through 08_CONTEXT) is just a **semantic container** for these layers.

If they're not separated, Claude loses context. Your team loses clarity. Investors don't see decision rationale.
